HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The footer element appears at the bottom of the page and usually contains things like the author, copyright, navigation etc
  • Other head (seen) elements: text elements, image elements, list elements.
  • Other body (unseen) elements: meta-related elements, content division elements, semantic elements.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A . preceding class designates it a class selector therefore any HTML element assigned to the class card will gain its property assignments from CSS.
  • Most of the basics are quite straight forward e.g. value is something like a specific colour or font and a property is something we're selecting to apply a value to.
  • External CSS style sheets are preferred as it allows developers to keep all their css rules to a single file which is easier to manage

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: adds the changed files from your working directory to the staging area of git thus "staging your changes" (add -A means to commit all files in the working branch)
  • git commit -m: commits changes that have been staged to the repository (-m flag is used to associate a message with the commit).
